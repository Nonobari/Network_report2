SRIE > SQL Injection > Description

SQL Injection is a type of attack that allows the attacker to extract database information from the web application. When it occurs, the attacker can access the database and extract information such as usernames, passwords, email addresses, etc. In worst cases, the attacker can even modify the database, which can lead to a complete compromise of the web application.

For example, if a website make a query like :
{% raw %} SELECT * FROM users WHERE username = 'input_username' AND password = 'input_password'; {% endraw %}
and the user input is not sanitized, the attacker can inject a SQL query to bypass the authentication.

For example, if the user input is admin' --, the query will become:
{% raw %} SELECT * FROM users WHERE username = 'admin' --' AND password = 'input_password'; {% endraw %} The -- is used to comment the rest of the query. The query will return the first user in the database, which is usually the administrator.

The attacker can then login as the administrator without knowing the password.
