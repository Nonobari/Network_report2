SRIE > SQL Injection > Description

L'injection SQL est un type d'attaque qui permet à l'attaquant d'extraire des informations de base de données de l'application Web. Lorsque cela se produit, l'attaquant peut accéder à la base de données et extraire des informations telles que des noms d'utilisateur, des mots de passe, des adresses e-mail, etc. Dans le pire des cas, l'attaquant peut même modifier la base de données, ce qui peut conduire à une compromission complète de l'application Web.

Par exemple, si un site Web effectue une requête du type :
{% raw %} SELECT * FROM users WHERE username = 'input_username' AND password = 'input_password'; {% endraw %}
et que l'entrée utilisateur n'est pas nettoyée, l'attaquant peut injecter une requête SQL pour contourner l'authentification.

Par exemple, si la saisie utilisateur est admin' --, la requête deviendra :
{% raw %} SELECT * FROM users WHERE username = 'admin' --' AND password = 'input_password'; {% endraw %} Le -- est utilisé pour commenter le reste de la requête. La requête renverra le premier utilisateur dont le nom d'utilisateur est admin de la base de données.

L'attaquant peut alors se connecter en tant qu'administrateur sans connaître le mot de passe.
